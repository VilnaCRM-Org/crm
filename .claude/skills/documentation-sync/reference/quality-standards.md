# Documentation Sync Quality Standards

## Required Updates

Update docs when changing:

- Makefile target behavior.
- Test commands or mock infrastructure.
- Agent skill layout.
- Feature folder conventions.
- Public component or hook APIs.

## Quality Rules

- Keep commands copy-pasteable.
- Mention the smallest useful verification command.
- Keep examples aligned with actual repo paths.
- Do not document backend-only workflows in this frontend repo.
